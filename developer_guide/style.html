Coding Style#

All coding must follow PEP8 style.

Automated Formatting#

However PEP8 allows a significant amount of flexibility which leads to code that looks different from one developer to another and disrupts one’s focus on the content. Hence code is further styled and automatically formatted according to the strict rules enforced by Black to ensure consistency.

Black can be run within a terminal window in the root directory using the following command:

poetry run black src tests

All configuration for Black can be found in pyproject.toml.

Linting#

We also use Ruff for linting to enforce enforce Google-style docstring convention and some other linting features not covered by Black. It can also be run in a similar manner using the following command:

poetry run ruff src tests

All configuration for Ruff can be found in pyproject.toml.

Sorting Imports#

We use isort for automated sorting of imports. It can also be run in a similar manner using the following command:

poetry run isort src tests

All configuration for isort can be found in pyproject.toml.
